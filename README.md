# データベース正規化インタラクティブ教材

ブラウザで動作する「データベース正規化学習教材」のMVPです。  
非正規形から第3正規形までを対象とし、学習者がレコードを編集・追加・削除しながら、更新異常・挿入異常・削除異常を体験できるようにします。

## 概要

この教材は、データベースの正規化をブラウザ上で学ぶためのインタラクティブ教材です。

正規化を定義暗記ではなく、データを実際に操作したときに起きる不整合や管理の難しさから理解することを目的としています。

## 学習目標

- 非正規形の問題点を理解する
- 第1正規形の意味を理解する
- 第2正規形の必要性を理解する
- 第3正規形の必要性を理解する
- 正規化を「定義」ではなく「データを壊れにくくする設計」として理解する

## できること

- 非正規形、1NF、2NF、3NF の4ステージを収録
- 各ステージでサンプルテーブルを表示
- セル編集、行追加、行削除
- 「異常検出」による問題点の表示
- 問題セルのハイライト
- ステージごとのミッション表示
- 「次の正規形へ」で分解後の段階へ遷移
- 3NF で異常が起きにくくなる構造を体験

## セットアップ

### 1. nvm を使う

このプロジェクトは `nvm` で Node.js を管理する想定です。  
リポジトリ直下の `.nvmrc` に推奨メジャーバージョンを定義しています。

```bash
nvm install
nvm use
```

`nvm` が未導入の場合は、先に `nvm` をインストールしてください。

### 2. 依存関係を入れて起動

```bash
nvm use
npm install
npm run dev
```

## ビルド

```bash
nvm use
npm run build
```

## GitHub Pages 公開

このプロジェクトには GitHub Pages 用の Actions workflow を含めています。

1. GitHub に新しいリポジトリを作成する
2. このプロジェクトを `main` ブランチへ push する
3. GitHub の `Settings > Pages` で `Build and deployment` を `GitHub Actions` にする
4. `main` への push 後、Actions が `dist/` を Pages に公開する

ローカル確認後に公開する流れ:

```bash
nvm use
npm run build
npm run dev
```

## 補足

MVPでは、各正規形で典型的に起きやすい異常をルールベースで検出しています。  
完全なDB設計検証ツールではなく、学習体験を優先した教材です。
